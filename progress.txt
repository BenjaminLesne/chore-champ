## Codebase Patterns
- Use `nuqs` with `parseAsStringLiteral` for type-safe URL enum params; wrap tests with `NuqsTestingAdapter` from `nuqs/adapters/testing`
- Insights page: server component (`page.tsx`) fetches data, client components render charts
- Card styling: `rounded-xl border border-gray-200 bg-white p-4 shadow-sm`
- Toggle button styling: `inline-flex rounded-lg bg-gray-200 p-1` with active state `bg-white text-gray-900 shadow-sm`
- Chart colors: COLORS array in `insights-charts.tsx` — reuse for consistency
- DB query functions go in `queries.ts`, pure testable logic in `period-utils.ts` — vitest browser mode can't import Node.js DB modules
- `getMemberDailyStats(householdId, period)` returns avg/stddev per member; `getMemberChoreBreakdown(householdId, period)` returns chore counts per member
- Export types (`MemberDailyStat`, `ChoreBreakdownEntry`) from `queries.ts` for UI components
- Tests use vitest browser mode with `vitest-browser-react` render
- Pre-existing typecheck errors in scoreboard.test.tsx and past-winners.test.tsx (missing props) — not introduced by this work
- Recharts `Cell` is deprecated — put `fill` in data objects; Recharts auto-reads the `fill` field
- `getMemberChoreBreakdown` returns a `Map<number, ...>` — convert to plain `Record<string, ...>` for client component serialization
- Recharts horizontal bar chart: use `layout="vertical"`, `XAxis type="number"`, `YAxis type="category"`, round bars with `radius={[0, 6, 6, 0]}`
- eslint `no-unsafe-type-assertion`: avoid `as` casts — use explicit typed props with optional fields + defaults
- `page.tsx` reads `?period=` search param and passes it to query functions for member stats sections
